Application Lifecycle Standards (sample)

Lifecycle states: plan, build, run, retire.

plan   — proposed or in business-case stage; no production footprint.
build  — under active development or implementation.
run    — in production and operationally supported.
retire — scheduled for or undergoing decommissioning.

Transition rules
- Movement is normally forward: plan -> build -> run -> retire.
- Re-entering 'build' from 'run' requires a documented change rationale.
- 'retire' is terminal; an application leaving 'retire' must be re-justified.

Technical maturity (1..5)
- 1 = legacy / unsupported technology; 5 = current, strategically aligned stack.
- Applications at maturity 1-2 in 'run' should have a modernisation or retirement
  plan recorded.
