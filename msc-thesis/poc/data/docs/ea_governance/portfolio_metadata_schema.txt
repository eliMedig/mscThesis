Application Portfolio Metadata Schema (sample)

Each portfolio record describes one application with these attributes:

app_id               APP-NNNN, unique, immutable.
app_name             Human-readable, unique.
owner                Accountable person/role. Mandatory.
lifecycle_state      plan | build | run | retire.
business_criticality low | medium | high | critical.
technical_maturity   integer 1..5.
last_reviewed_date   ISO date of the last governance review.
compliance_status    compliant | non_compliant | unknown.
hosting              on_prem | cloud | hybrid.
annual_cost          estimated annual run cost (currency units).

This schema is the contract the anomaly-detection pipeline reads. Rule-based
checks operate on these fields directly; the statistical model encodes the
categorical fields and scores cross-feature patterns.
